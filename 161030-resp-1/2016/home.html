[Lucene.net] 2. Your First Lucene.net Application

We learned what Lucene.net is in the previous tutorial. Let’s make a lucene.net application.

What you will learn from this tutorial

  • How to use basic classes of Lucene.net

What Will We Make?

We are going to make simple and basic console application which uses Lucene.net.

Add Lucene.net To Your Project

  1. Turn on Visual Studio and make a console project. Name it FirstLucene or whatever you want.
  2. Right button click the project name to show the context menu and click Manage NuGet Packages.
  3. Then search Lucene.net, using the search box on the right top.
  4. Finally, click “install.” Then, Lucene.net is added to your project.

Program

A little ago, we added Lucene.net to our project. Let’s write some code!

Main Program

This is the main program. We will use the classes Indexer and Searcher we made for this project.

class Program
{
    static void Main(string[] args)
    {
        string indexDir = "indices";
        using (Indexer indexer = new Indexer())
        {
            indexer.IndexDirectory = indexDir;
            indexer.DataDirectory = "data";
            indexer.Setup();

            indexer.Index();
        }

        Searcher searcher = new Searcher();

        searcher.Search(indexDir, "Sonic");

        Console.WriteLine();

        searcher.Search(indexDir, "Bible");

        Console.WriteLine();

        searcher.Search(indexDir, "colors");
    }
}

First you should see the indexDir string on line 5. We will store original documents in “data” folder and indices in “indices” folder. If you downloaded the code from the site, then you should copy “test/data” folder to “bin/Debug” or “bin/Release” folder. In the “data” folder, there are some text file copied from Wikipedia.

This code is separated into 2 parts, indexing(line 6-13) and searching(line 15-25). Method names are self-exploratory, so I don’t explain them here, we will see what is going on in these methods soon.

Indexer

using Lucene.Net;
using Lucene.Net.Store;
using Lucene.Net.Index;
using Lucene.Net.Analysis.Standard;
using Lucene.Net.Util;
using LVersion = Lucene.Net.Util.Version;
using Lucene.Net.Documents;

public class Indexer : IDisposable
{
    public String IndexDirectory { get; set; }
    public String DataDirectory { get; set; }

    public Indexer()
    {
    }

    public void Setup()
    {
        Directory dir = FSDirectory.Open(IndexDirectory);
        writer = new IndexWriter(dir, new StandardAnalyzer(LVersion.LUCENE_30), IndexWriter.MaxFieldLength.UNLIMITED);
    }

    public void Dispose()
    {
        writer.Dispose();
    }

    public int Index()
    {
        String[] files = System.IO.Directory.GetFileSystemEntries(DataDirectory);
        foreach (String name in files)
        {
            IndexFile(name);
        }
        return writer.NumDocs();
    }

    private void IndexFile(string name)
    {
        Document document = GetDocument(name);
        writer.AddDocument(document);
    }

    private Document GetDocument(string name)
    {
        Document d = new Document();

        System.IO.StreamReader r = new System.IO.StreamReader(name, System.Text.Encoding.Default);
        string text = r.ReadToEnd();

        d.Add(new Field("content", text, Field.Store.NO, Field.Index.ANALYZED));
        d.Add(new Field("filename", name, Field.Store.YES, Field.Index.NOT_ANALYZED));
        System.IO.FileInfo info = new System.IO.FileInfo(name);
        d.Add(new Field("fullPath", info.FullName, Field.Store.YES, Field.Index.NOT_ANALYZED));

        return d;
    }

    private IndexWriter writer;
}

First of all, every Lucene.net class is located under Lucene.Net namespace or its subnamespace, so I added many “using namespace” declaration on the top.

The most important function of this class in Index() function(line 29-37). This function retrieves the list of file names in the “data” folder(line 31) and creates the index of the documents(line 32-35), and returns the count of indexed documents.

Before we read the code of IndexFile() function, let’s look at the SetUp() function. This function opens the FSDirectory(FS here stands for File System) from the path where the index is, and creates IndexWriter class. This IndexWriter class writes indexes to the Directory(we used FSDirectory here, and there are many other directories, I will cover them later).

To create IndexWriter, you need 3 parameters, Directory instance, analyzer instance, and MaxFieldLength enum value. We will see each in detail later, but we need to take a look at LUCENE_30 value. Lucene.net tries to be as backward-compatible as possible, and if there are serious bugs in previous versions and cannot ignore them because of the backward compatibility, then Lucene.net developers try to emulate that bug, so there is a Version Parameter. One more thing you should notice is the using LVersion = Lucene.Net.Util.Version; in the namespace declarations. Since there is a System.Version class, we cannot use Lucene.Net.Util.Version directly, because it make the name Version ambiguous.

IndexFile() function creates document from GetDocument() function and calls AddDocument method of IndexWriter.

GetDocument() function creates a Document instance and add Field instances to the document. I added content, filename, fullpath information to the document. I will cover the options in the parameters of the Field constructor later.

Searcher

using Lucene.Net;
using Lucene.Net.Store;
using Lucene.Net.Index;
using Lucene.Net.Analysis.Standard;
using Lucene.Net.Util;
using LVersion = Lucene.Net.Util.Version;
using Lucene.Net.Documents;
using Lucene.Net.Search;
using Lucene.Net.QueryParsers;

public class Searcher
{
    public void Search(string indexDir, string query)
    {
        using(Directory dir = FSDirectory.Open(indexDir))
        using(IndexSearcher searcher = new IndexSearcher(dir))
        {
            QueryParser parser = new QueryParser(LVersion.LUCENE_30, "content", new StandardAnalyzer(LVersion.LUCENE_30));

            Query q = parser.Parse(query);
            TopDocs hits = searcher.Search(q, 10);

            Console.WriteLine("Found " + hits.TotalHits + " documents matched query '" + query + "':");

            foreach (ScoreDoc d in hits.ScoreDocs)
            {
                Document doc = searcher.Doc(d.Doc);
                Console.WriteLine(doc.Get("filename"));
            }
        }
    }
}

There are 5 classes that we should look closely.

IndexSearcher

This class searches the index. By using Search method with the Query instance, we can search what we want from the index.

QueryParser

Generally, users input queries in strings, and QueryParser parses the string and make it into a Query instance. To make this transformation, QuaryParser needs Analyzer, too. Normally, the analyzer in QueryParser class and IndexWriter are same. If you use different analyzers, then it can cause a subtle problem, so if you don’t know much about Lucene.net, then use the same one.

Query

Query class shows what the user want. This can be made progmatically or by QueryParser, or you can use both.

TopDocs

This class holds the most relevant result. You can get the result count by using TotalHits property and list of relevant result by ScoreDocs property.

ScoreDoc

This class provides the access to each search result in TopDocs. You can get the document by calling IndexSearcher.Doc() method.

result

There Is No Foolish Question

Q: I wrote the code like below and there is no result. What happened?

// To run this program properly, please read ReadMe.txt
class Program
{
    static void Main(string[] args)
    {
        string indexDir = "indices";
        using (Indexer indexer = new Indexer())
        {
            indexer.IndexDirectory = indexDir;
            indexer.DataDirectory = "data";
            indexer.Setup();

            indexer.Index();

            Searcher searcher = new Searcher();

            searcher.Search(indexDir, "Sonic");

            Console.WriteLine();

            searcher.Search(indexDir, "Bible");

            Console.WriteLine();

            searcher.Search(indexDir, "colors");
        }
    }
}

A: IndexWriter doesn’t write indices to the file system when you called the AddDocument method. It flushes them when Commit() or Dispose() method is called. Moreover, IndexSearcher class cannot search the documents which are created after it was opened, so that problem happened.

[Lucene.net] 1. What Is Lucene.net?

Before we start to write codes with Lucene.net, let’s learn about Lucene.net.

What you will learn from this tutorial

  • What Lucene can do and cannot do.
  • Components of Search Engine Software
  • Relation Between Lucene and Lucene.net

What Is Lucene?

In short, Lucene is a search library. It is not a search engine software like Google or Bing. It doesn’t provide you some fabulous search interface, and it doesn’t help you collect documents. It just helps you search text. Specifically, Lucene helps you index documents and search those indexed documents.

Components of Search Engine Software

Behind the simple search box, there are many complicated components. The blue components in the picture are supported features of Lucene and white ones are not.

Indexing

At the end of the almost every book, there are index pages. If you want to find something in the book, you can open the index and find the word you want and go to the page written next to the word. Index in search engine does the same thing. You cannot open every single document when it is searched, because it will cost tremendous time. To save the time, we have to make indices.

Acquire Content

To index content, you have to get the contents to be searched. This sounds pretty natural and easy, but this acquisition is sometimes really hard. Suppose that you want to make a IT blog search engine. Then you should visit IT blogs, finding IT blogs is a hard work and you need to distinguish IT blogs from other blogs. These kinds of requirements are sometimes pretty demanding.

Build Document

After you get some contents, you need to extract what you want from the content. If it is a blog post, then you should extract title, writer, content, date, etc. As you will see later, you will add these fields to a document, and Lucene finds these documents.

Another common thing you should do is boosting the field or document. When you build the document, you should decide which document or field is important, this boosting can affect the search result and quality a lot, so you should research hard to make a good boosting algorithm.

Analyze Document

When analyzing a document, words in the document should be changed into a token and some words should be deleted.

Typically, we should find documents which contains the word ‘dying’ if a user searches ‘die.’ ‘Die,’ ‘die,’ ‘dying,’ ‘died,’ ‘dead,’ etc. should be changed into a token so those word can be treated as a one word, because the user expects that.

Some words like to, an, the are not important and frequently appear in the content. But they have no meaning, so they will not be searched frequently. To save the disk space and get more speed, we should ignore those words.

Index Document

After these hard processing, the document is added to the index. Lucene covers this part, so we have nothing to worry about this. After indexing, the document is ready to be searched.

Searching

Now we go back to our topic, search. Indexing has its own interesting part, but we only need indices because we cannot search fast without it.

User Interface

We will not cover search user interfaces in this tutorial series, but this is the most important part. If you have a brilliant indexer and the greatest searcher behind the scene but the UI looks bad, then users will think that your application is bad.

Here are some useful tips for search user interfaces.

  • Keep the user interface simple. Don’t show much advanced options on the first page.
  • Make the search box, ubiquitous, easy to find, visible everywhere. Don’t let your to click to start searching.
  • Result representation is also important. Highlight keywords, do not use small fonts, do not cram too much text.
  • Default search order should be clear(usually, relevance).
  • Be fully transparent. If your search result does something interesting, please show them to the user(e.g. If a user searches “sonic amature games expo” and your search engine include the amateur in the result, then show the message, “Including results for sonic amateur games expo” like Google). If you are not transparent about your search result, then users will not trust your search result, and winning their trust back is almost impossible.
  • Dog food your search result. Enjoy what is good, and correct everything that is bad.
  • Research what Google does if you don’t know what to do.

Build Query

If a user enters a query, then the query is also analyzed and parsed into query classes. Lucene.net’s QueryPaser class does the job.

Run Query

After building a query, we should find the document that matches the query. Lucene does this and there are many extension points to meet your needs. We will cover these later.

Result

After running query is done, the results are returned. You should show them to the user.

And The Rest

Admin Interface

To use Lucene better, you need to control its RAM, file descriptor usage, etc, and you can start and stop the server with this interface. Admin interface helps the administrator operate the search engine better.

This is trivial if you are making a desktop search engine, but if it is an enterprise search engine, then it can be one of the most important requirements.

Analytics Interface

To operate search engine better, you should research your users, and this research is done by studying the query, result patterns. Good search engine analytics interface should include:

  • How often which kinds of queries are run
  • Queries that hit low relevance
  • Queries where the user didn’t click on any results
  • How often users are sorting by specified fields instead of relevance
  • The breakdown of Lucene’s search time
  • Indexing metrics: documents indexed per second, byte size of documents being indexed.

What is Lucene.net?

Lucene was originally written in Java by Doug Cutting. He is a famous opensource creator. In addition to Lucene, he created Nutch, an opensource web search engine, and Hadoop, open-source software for reliable, scalable, distributed computing.

As Lucene gets its fame, many other language users want add Lucene to their program, so the Lucene is ported to many other languages. Lucene.net is one of them and written in C#.

The developers of Lucene.net has 3 primitive goals like below:

  1. Maintain the existing line-by-line port from Java to C#, fully automating and commoditizing the process such that the project can easily synchronize with the Java Lucene release schedule;
  2. Maintaining the high-performance requirements expected of a first class C# search engine library;
  3. Maximize usability and power when used within the .NET runtime. To that end, it will present a highly idiomatic, carefully tailored API that takes advantage of many of the special features of the .NET runtime.

Lucene.net graduated from the Apache Incubator on August 15th, 2012.

Lucene.net web site is http://lucenenet.apache.org/ and Lucene.net team blog is https://blogs.apache.org/lucenenet/.

There Is No Foolish Question

Q: I want to search my image files with Lucene. Is that possible?

A: Yes or no. As I said before, Lucene is a text search engine, so if you can extract some text from the image, then you can use Lucene to search that text.

Q: I should make a search engine for my team, but Lucene or Lucene.net is too low level. I need to make so much things. Are there any other options?

A: There is Solr, sister project of Lucene. This project is based on Lucene and provides many server-side features, so if your project is server-client based, then it will save you a lot of time.

Lucene.net Tutorial List

Lucene.net is a C# port of the famous search library Lucene. The developers of Lucene.net almost translated this library from Java to C#. But there are some differences between those libraries, because C# has many advanced features. They tried to make it look more C#-like. These differences are sometimes trivial(e.g. just changing getters and setters to properties), but there are some significant changes(e.g. changing close() functions to Dispose() functions to use “using” keyword). Thanks to these changes, I decided to write this tutorials.

This tutorial tries to cover many aspects of Lucene.net.
